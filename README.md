# Deadline Bot

Telegram-бот для дедлайнов учебной группы. Бот принимает дедлайны только от пользователей из whitelist, публикует и обновляет посты в канале или треде супергруппы, хранит историю изменений и поддерживает архив.

## Быстрый старт

Нужен Python 3.11+.

1. Установите зависимости:

```bash
pip install -r requirements.txt
```

2. Заполните `.env`:

```env
TOKEN=ваш_токен_бота
CHANNEL_ID=-100xxxxxxxxxx
CHANNEL_THREAD_ID=
WHITELIST_USER_IDS=123456789,987654321
DEADLINES_STORAGE_PATH=/абсолютный/путь/к/deadlines.json
INCLUDE_DEADLINE_HASHTAG=1
```

`CHANNEL_THREAD_ID` можно оставить пустым. Тогда бот будет писать в общий чат супергруппы, а не в конкретный topic/thread.
`INCLUDE_DEADLINE_HASHTAG=1` включает футер вида `🔜  #дедлайн` в каналных сообщениях. Если поставить `0`, `false`, `no` или `off`, этот футер исчезнет из шаблонов.

3. Запустите бота:

```bash
py app.py
```

## Интерфейс

В основной клавиатуре только три кнопки:

- `Список дедлайнов`
- `Архив`
- `Обновить посты`

Пользовательский интерфейс бота работает только в личке. Сообщения и команды в группах или супергруппах бот игнорирует; туда он пишет только собственные посты о дедлайнах.

### Список дедлайнов

Это основной рабочий экран.

- список постраничный, поэтому не упирается в лимит длины сообщения Telegram
- в список попадают `active`, `cancelled` и `completed` дедлайны
- `cancelled` и `completed` остаются в списке, пока бот не удалит их сообщения через 3 дня и не перенесёт запись в архив
- из списка можно открыть карточку конкретного дедлайна
- на экране списка есть inline-кнопка `Добавить дедлайн`

### Карточка дедлайна

В карточке показываются основные данные дедлайна и кнопки действий.

Для активного дедлайна доступны:

- `Изменить`
- `Напомнить`
- `Отменить`
- `Удалить`
- `Подробности`

Для завершённого или отменённого дедлайна, пока он ещё в основном списке, остаются:

- `Удалить`
- `Подробности`

### Подробности

Экран подробностей показывает:

- описание
- статус
- дату и время
- время создания
- автора
- число сообщений в канале
- историю событий по дедлайну

История включает создание, публикации, напоминания, изменения, отмену, завершение, удаление и перенос в архив.

### Архив

Архив тоже постраничный. Из него можно открыть карточку дедлайна и подробности, но архивные записи уже не редактируются и не публикуются заново автоматически.

## Создание и изменение дедлайна

Формат даты:

- `DD.MM.YYYY`
- `DD.MM.YYYY HH:MM`

Примеры:

```text
ИДЗ по матану
13.04.2026
```

```text
ИДЗ по матану
13.04.2026 18:30
```

Правила:

- все расчёты идут в часовом поясе `UTC+5`
- если время не указано, логически используется `00:00`
- если время не указано, в тексте дедлайна оно не выводится
- если пользователь явно ввёл `00:00`, бот это показывает
- форматирование описания сохраняется через Telegram HTML, включая `bold`, `italic`, `blockquote` и кастомные emoji

При изменении дедлайна:

- можно пропустить изменение описания
- можно пропустить изменение даты
- если фактически ничего не изменилось, бот не фиксирует изменение
- если по дедлайну уже были публикации, бот отправляет в канал отдельное сообщение о смене дедлайна

## Публикации и напоминания

Логика публикации такая:

- если до дедлайна 7 дней или меньше, бот публикует его сразу
- если до дедлайна больше 7 дней, бот спрашивает, делать ли первую публикацию сразу
- затем бот напоминает за 7 дней и за 24 часа
- ручная кнопка `Напомнить` использует тот же шаблон, что и автоматические напоминания
- при каждом новом сообщении о дедлайне старые сообщения этого дедлайна удаляются и заменяются одним свежим постом
- уже опубликованный активный пост автоматически обновляется в фоне, поэтому `Осталось дней / часов / минут` не зависает на старом значении

Когда наступает момент дедлайна:

- бот не шлёт новый пост
- бот удаляет лишние старые сообщения этого дедлайна, если они остались, и помечает последнее сообщение как завершённое

При отмене:

- бот сначала спрашивает причину отмены
- затем публикует зачёркнутое сообщение дедлайна с подписью `Отменён по причине: ...`
- через 3 дня удаляет все сообщения этого дедлайна из канала

При удалении:

- бот сразу удаляет все сообщения дедлайна из канала
- запись уходит в архив немедленно

## Обновление старых постов

Кнопка `Обновить посты` и команда `/refresh_posts` заново рендерят все сохранённые сообщения канала по текущим шаблонам из [bot_messages.py](C:/Users/Roma/Desktop/projects/deadline%20bot/bot_messages.py).

Это нужно, если вы поменяли:

- тексты
- HTML-разметку
- кастомные emoji
- общий вид постов
- включение или выключение футера `🔜  #дедлайн`

## Хранение данных и миграции

Бот хранит состояние в JSON-файле:

- если задан `DEADLINES_STORAGE_PATH`, используется он
- иначе используется локальный `deadlines.json`

Структура файла версионируется через `schema_version`. При запуске бот:

- читает текущий JSON
- при необходимости автоматически прогоняет миграции
- переписывает файл уже в новом формате
- если файл битый или имеет неподходящую структуру, сохраняет его рядом как `unformatted-<имя>.json`, `unformatted-<имя> (1).json` и так далее, а затем создаёт новый пустой storage

То есть серверный файл данных не нужно вручную чистить при каждом изменении схемы. Если схема меняется несовместимо, для неё нужно добавить одноразовую миграцию в [app.py](C:/Users/Roma/Desktop/projects/deadline%20bot/app.py).

## Команды

- `/start` — показать основную клавиатуру
- `/new` — создать дедлайн
- `/list` — открыть основной список дедлайнов
- `/archive` — открыть архив
- `/refresh_posts` — обновить сохранённые сообщения канала по текущим шаблонам
- `/now` — показать текущее время глазами бота
- `/cancel` — отменить текущий сценарий ввода

## Утилита `tools.py`

Запуск:

```bash
py tools.py
```

Это отдельный вспомогательный бот, который использует тот же токен. Его не нужно запускать одновременно с `app.py`.

Сейчас он умеет:

- `Debug Private Messages`
- `Debug Public Chats`
- показывать `text_html`
- показывать Telegram entities
- извлекать `custom_emoji_id`
- показывать `message_thread_id`
- показывать `is_topic_message`

Это удобно для работы с кастомными emoji, HTML-разметкой и тредами супергрупп.
